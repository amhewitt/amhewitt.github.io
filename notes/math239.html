What is a graph?

A graph is a set of vertices V, and a pair of edges E. An edge is a set of two vertices. The two vertices in an edge are said to be connected.
When represented visually, a vertex is a point, while an edge is a side that connects two points together.

Molecules are often represented as graphs. Atoms are vertices, and the chemical bonds between them are edges. You might also represent a map of the world via a graph. Each vertex might be a country, and two countries are adjacent if they share a border.

Other examples:

  • Human connection (eg. Facebook)
  • Radio stations: each radio station is a vertex. Each edge represents two radio stations that have the same frequency, and thus will interfere if they are too close.

A vertex can also be conceptualized as a node. We will use the terminology vertex for this course.

Oftentimes, graphs come up in unusual ways. There is another way to label vertices and edges to get the same picture. Vertices can also be represented as things like 2-subsets of a certain range of integers, etc. Other examples will be apparent as the course continues.

Two vertices are said to be adjacent if they are joined by an edge. In the edge {1, 2}, the vertices 1 and 2 are adjacent.

When are two graphs the same? When are they different?

There are two ways to look at graphs. If labelling matters, then you might say that two otherwise identical graphs are different if they do not use the same vertices labels. If it does not, you might say that as long as there exists a bijection from one set of vertices to another, then they are expressing the same graph.

Two graphs (V1, E1) and (V2, E2) are isomorphic if there is a bijection F: V1 → V2 such that {u,v} exist in E1 if and only if {f(u),f(v)} exist in E2.

For this course, saying two graphs are isomorphic is tantamount to saying that they are two different representations of the same graph.

Can we tell if two graphs are isomorphic?

Say we have a graph G = (V,E). V = {1,2,3,4,5}, and E = {{1,2}, {2,3}... {5,1}}, and another graph H where V(H) = {1,2,3,4}.

We can instantly tell that G and H are not isomorphic without even knowing anything else about H, because G has 5 vertices, while H only has 4. This follows from the definition of a bijection - two isomorphic graphs must have the same number of edges and vertices.

The degree of a vertex v is the number of edges that contain v. deg(v) is shorthand for the degree of v.

If F: G → H is an isomorphism, then for every vertex v of G, degG(v) = degH(f(v)).

To show that two graphs are isomorphic:

  • Exhibit a bijection
  • Show that the bijection preserves the edges (provide a argument for this)

To show that a bijection does not hold, show that the bijection being proposed does not preserve the edges.

Cycles

For k ≥ 3, a k-cycle is a sequence (v0, v1, ... , vk-1, v0) of vertices, all distinct, such that for each i = 1, 2, ..., k, (vi-1, vi) is an edge.

Generally speaking, the direction and order of the sequence does not matter, but there will be instances where it does matter.

(You can't have a 2-cycle or a 1-cycle, for obvious reasons. These are not defined.)

If G has an odd-length cycle, then there is no 2-colouring of the vertices so that every edge joins vertices of different colours.

In other words, if G has a 2-colouring of the vertices so that every edge joins vertices of different colours, then G has no odd-length cyles.

Bipartite Graphs

The first part of the above fact also describes a bipartite graph.

A graph G is bipartite if it has a 2-colouring of its vertices so that every edge joins vertices of different colours.

Paths

A k-path is a sequence (v0, v1, ... , vk) of distinct vertices such that, for i = 1, 2, ..., k, (vi-1,vi) is an edge.

k path = path of length k, a path with k edges. (A k path will have k + 1 vertices). A 0-path is just a vertex (and indeed, every vertex is a 0-path). A 1-path is just an edge (and indeed, every edge is a 1-path).

Walks

A walk in a graph is a sequence (v0, v1, ..., vk) of vertices in G such that for each i = 1., 2., ..., k, (vi-1,vi) is an edge of G.

Note that a path is a walk with all vertices distinct.

A walk is closed if it comes back to where it starts, in essence, v0 = vk.

Also, note that a cycle is a closed walk such that (except for the beginning being the same as the end), all vertices are distinct.

Connectedness

A graph is connected if there is a walk in G that contains all the vertices. Or, for any two vertices u and v in G, there is a path in G from u to v.

Trees

A tree is a connected graph with no cycles.

A forest is a graph with no cycles (named such because every component of a forest is a tree).

Every tree is a forest, and every forest contains at least one tree.

In addition, every path is a tree. Except for the smallest tree (just a single vertex), every tree has a vertex of degree 1.

An Euler Tour is a closed walk traversing every edge exactly once.

Theorem: A graph G has an Euler tour iff G has only one component with edges, and every vertex of G has even degree.

Proof: Suppose G has an Euler tour (v0,e1,...,ek,v0). If e and f are edges of G, then there are indices i and j such that e = ei and f = ej. The walk (vi-1,ei,vi,ei+1...vj-1,ej,vj) shows that the ends vi-1,v1 of e and the ends vj-1,vj are in the same component of G.

Therefore e,f are in the same component of G.

If we consider a vertex of G in that component (the others have degree 0, which is even), we have v = vi1...vij for some indices 0 ≤ i1 < ... < ij ≤ k . The v is incident with the edges ei1,ei1+1...,eij,eij+1.

What if ei1+1 = ei2? Because vi1+1 ≠ vi1, the index i2 ≠ i1+1. The same holds all along. Thus, the edges listed are all different. They come in pairs, so there is an even number of them.

Now, suppose G is a connected graph and every vertex of G has even degree. We want to prove G has an Euler tour. Start at a vertex v. We prove this by induction on |E(G)|.

If |E(G)| = 0, then G, being a connected graph, has one vertex v; (v) is the tour.

Now suppose |E(G)| > 0. Then every vertex has positive degree. Since every vertex has even degree, every vertex has degree ≥ 2. By previous theorem, G has a cycle C. Delete the edges of C to get a smaller graph with fewer edges. What happens?

Let H be the subgraph of G obtained from G by deleting the edges of C. Every vertex of H has even degree (same as G or two less, if in C). Also, |E(H)| < |E(G)|(H has fewer edges).

More In-depth Discussion of Trees

Recall: a tree is a connected graph with no cycles. We saw that if a graph has every vertex with degree at least 2, then the graph has a cycle. Then, every tree has at least one vertex with degree 1, by corollary. (The exception is the trivial tree that has no edges and only one vertex.)

Every path is an example of a tree, and it has two vertices of degree 1.

Let a tree T exist, where all vertices except one, u, has at least degree 2. What do we get when look at T - u?

Fact: if T is a tree and u is a vertex with degree 1 in T, then T - u is a tree with one less vertex than T.

If you delete a vertex with degree larger than 1, then you end up with multiple components, all of which are trees.

Facts about trees:

  • If T is a tree with nvertices, then |E(T)| = n-1. (|V(T)| - |E(T)| = 1)
  • If T is a tree, then T is bipartite. (very important!)

Bridges

A bridge is an edge e in a graph G where G - e has more components than G. Alternatively, an edge e in a graph G is a bridge if, for K the component of G containing e is such that K - e is not connected.

When is an edge not a bridge? If an edge is in a cycle, it will never be a bridge.

Converse is also true: if e is not a bridge, then e is in a cycle.

Theorem: An edge e of a graph G is NOT a bridge if and only if e is in a cycle of G.

By consequence, every edge in a tree is a bridge.

If every vertex of G has even degree, then G has no bridge, since G then has an Euler tour. Then every edge is in a cycle.

Spanning Trees

A subgraph H of a graph G is spanning if it includes all of G's vertices.

A spanning tree in a graph G is a spanning subgraph that is a tree. (That is, a subgraph that contains all of G's vertices, is connected, and has no cycles.)

When does a graph have a spanning tree? Theorem: a graph G has a spanning tree if and only if G is connected.

Proof: suppose G has a spanning tree T. Let u,v be vertices of G. Since T is spanning, u,v ∈ V(T). Since T is a tree, there is a uv-path in T. A T is a subgraph of G and P is a subgraph of T, P is a subgraph of G. Therefore G has a uv-path, so G is connected.

Now suppose G is connected. If G has a cycle C, then an edge e of C is not a bridge of G. Therefore, G-e is a smaller connected spanning subgraph of G. By induction on the number of cycles, G-e has a spanning tree, so G has a spanning tree.

Weight

The weight of an edge is an arbitrary value assigned to an edge. We are interested in building spanning trees of a graph that minimizes the sum of the weights of the edges in the tree.

In order to create a spanning tree that minimizes the weight, the edge to pick must be in different components of our current forest (no cycle rule), and then pick the edge that follows that rule with the least weighted edge.

This algorithm produces a spanning tree of small weight. But can we do better? Can we prove that this does or doesn't yield a minmium weight spanning tree?

Let's try to compare a spanning tree produced by the algorithm with another spanning tree that presumably has smaller weight - call such a tree S. We will prove S does not exist.

Proof: Let F1...Fn = T be the sequence of forests obtained by the algorithm, and let S be a smallest weight spanning tree.

If S ≠ T, what we will do is find another spanning tree S' such that wt(S') ≤ wt(S) and S' has more edges in common with T than S does.

Algorithm for finding a minmium weight spanning tree:

  1. Start with a forest F, with all vertices, no edges
  2. Given Fi with i < n-1 edges, obtain Fi+1 from Fi by adding an edge of least weight that does not introduce a cycle (or, in other words, the new edge has its ends in different components of Fi)

Theorem: this algorithm outputs a minmium weight spanning tree.

Proof: Let the edges picked for the output tree be, in order, e1, e2,...,en-1. We show by induction that for each i, there is a minmium weight spanning tree containng all of e1, e2,...,ei.

Let Si be a minmium weight spanning tree containing e1...ei. If ei+1 ∈ E(Si), then set Si+1 = Si and this contains e1...ei+1 as required.

Interesting case: ei+1 ∉ E(Si). The subgraph Si + ei+1 is connected, not a tree, so it has a cycle C. Let f ∈ E(C) and let Si+1 = (Si + ei+1) - f. Planar Graphs

A graph G is planar if it can be drawn in the plane so that no two edges cross.

Which graphs are planar?

Not all graphs are planar - the Petersen graph is an example of a graph that is not planar. No matter how you try to represent it in the plane, you have edges that cross.

Every tree is planar, and therefore every forest is planar, since a forest is a series of trees.

A graph is planar if and only if each of its components are planar.

The complete graph notated as Kn is the graph with n vertices, with every pair having an edge joining them.

We can see that K1, K2, K3, K4 are all planar. However, K5 is not.

The complete bipartite graph, notated as Kp,q is the graph where the vertices partition into a set of size p and one of size q, and where everyone every one one of the p is adjacent to every one of the q.

K1,n is a tree, so it will always be planar, Likewise, K2,n is also planar for any n. K3,3 is not planar.

As far as cubes go, in an n-dimensional space (Qn), you want to look at all the spaces that have coordinates 0 or 1. In an n-cube, the vertices are all binary strings of length n. Edges join two vertices that differ in exactly one coordinate. Q3 is planar. Q4 is not.

The tetrahedron, the 3-dimensional cube (3-regular, 4-cycles, 6 faces), the octahedron (4-regular, 3-cycles, 8 faces), the dodecahedron (3-regular, 5-cycles, 12 faces), and the icosahedron (5-regular, 3-cycles, 20 faces) are all the Platonic solids, which are polygons that are n-regular for some integer n, are made entirely of k-cycles for some integer k, and can be represented by a planar embedding.

There is a linear relationship between the number of edges, vertices and faces. It is |V| + |E| = 2 + |F|. This is called Euler's formula.

The Jordan Curve Theorem underlies what distinguishes faces. If C is a simple closed curve in the plane R2, then R2 \ C has precisely two components. Any arc in R2 joining point in different components of R2 \ C must intersect C.

The degree of a face is the length of its boundary walk. Sum of the face lengths = 2 * number of edges. This is no coincidence, since you're traversing each edge twice.

This lends itself to a face-shake lemma. If G is embedded in the plane, then Σ f a face of C deg(f) = 2|E(G)|

A recap on Euler's Formula: Let G be a connected graph embedded in the plane with set F of faces. Then,

|V(G)| - |E(G)| + |F| = 2
↠ -|E(G)| = 2 - |V(G)| - |F|
↠ |E(G)| = |V(G) + |F| - 2

Proof: Our proof is by induction on the number of cycles in G.
Base case: G has no cycles. Since G is connected, G is a tree and so |E(G)| = |V(G)| - 1. In any planar embedding of G, there is only one face.
If G has more than 0 cycles, let e be an edge of G in a cycle C. One face of G is inside C, and the other is outside of C, so these are different faces of G. These faces become one face of G-e, so G-e has one fewer face than G. Since e is not a bridge, G-e is connected. By induction on # of cycles, |V(G-e)| - |E(G-e)| + |F(G-e)| = 2.

Colouring Graphs

Given a graph, colour the vertices of the graph with as few colours as possible, so that adjacent vertices get different colours.

An n-colouring of a graph is a colouring where n colours are used. It is not obvious when the least colouring is found. The chromatic number of a graph G, denoted Χ(G), is the smallest n such that G has an n-colouring.

Χ(G) = 1 if and only if G has no edge.

Χ(G) = 2 if and only if G has an edge and is bipartite. (Then partitions A and B will have their own colour.)

Χ(G) = 3 if and only if ... we have no idea... We don't know what makes a graph have a chromatic number of 3.

Algorithm: pick an ordering of vertices, then find the smallest available colouring going from left to right on the ordering. This is called a greedy colouring.

What is a good order for the vertices?

Lowest degree end rule: Find a vertex of smallest degree, put it at the end, then remove the vertex and all its associated edges. Then find the new lowest degree vertex and put it at the front of the list. Repeat. If all of the edges are the same degree, pick one at random. Keep track of the degreea of the vertices you delete as you delete them. Then the worst that can happen is the biggest number + 1 (representing the newest colour), as the degree represents the amount of vertices adjacent to that vertex that have already been coloured.

Critical point: if the biggest degree that happens is n, you know you are not going to need more than n + 1 colours for the whole graph.

We can prove that every planar graph has at most, a 6-colouring. This is because a planar graph will have degree at most 5 when picked out by the algorithm.

Matchings and Covers

A matching in a graph G is a set of edges M, such that each vertex of G is incident with at most one edge in M.

The empty set of edges is a matching in any graph. This is the smallest matching in a graph.

A perfect matching is a matching M in which every vertex is incident with exactly one edge of M.

Fact: For a graph G and a matching M, in G, the number of edges in M is at most (|V(G)| / 2).

Fact: A matching M in a graph is perfict if and only if the number of edges in M = (|V(G)| / 2). As a corollary, if there is a perfect matching, the number of vertices is even.

We use μ(G) to denote the size of a largest matching in a graph G.

μ(K6) = 3

μ(Cn) = ⌊n/2⌋ = μ(Kn)

A cover (of the edges by vertices) in a graph G is a set C of vertices such that every edge of G is incident with at least one vertex in C. They are kind of dual inverse concepts. The trivial cover for any graph is the entire set of vertices V(G). This is the largest cover any graph can have.

We use τ(G) to denote the size of a smallest cover of G.

τ(C8) = 4

τ(Cn) = ⌈n/2⌉
τ(Kn) = n - 1

τ(Kp,q) = max{p,q}

König's Theorem

For a bipartite graph G, μ(G) = τ(G).

Lemma: Let G be a graph, let M be a matching in G, and let C be a cover of G. The, |M| ≤ |C|.

Proof: For each edge uv of M, at least one of u and v is in C (because C is a cover).

Define f(uv) to be any vertex in C ∩ {u,v}. To see that f:M → C is one to one, we note that no vertex in C can be incident with more than one edge in M, because M is a matching. Therefore, |M| ≤ |C|. QED.

Corollary: let G be a graph, and let M,C be a matching and a cover in G. If |M| = |C|, then M is a maximum matching (largest) and C is a minimum cover (smallest).

Proof: let M' be any matching and let C' be any cover. By the lemma, then |M'| ≤ |C| and |M| ≤ |C'|.

Therefore, |M'| ≤ |C| = |M|, so |M'| ≤ |M|. Thus, M is a maximum matching.

M-Alternating Path

An M-Alternating path is a path of a graph G where every other edge is part of some matching M of G.

An M-Augmenting path is an M-alternating path that starts and ends at vertices not incident in G with any edge of M.

Theorem: if M is a matching in a graph G, and there is an M-augmenting path P, then M (circleplus) E(P) is also a matching and |M (circleplus) E(P)| = |M| + 1.

M (circleplus) E(P) consists of those edges of M not in P and those edges of P not in M. (Symmetric difference; (M ∪ E(P)) \ M ∩ E(P))

In the same vein, |C| ≤ |C'|, so C is a minimum cover.

A saturated vertex V in a matching M is a vertex such that at least one of the edges attached to V are in M.

An unsaturated vertex V in a matching M is a vertex such that none of the edges attached to V are in the matching.

Find a (minimum?) cover that is the same size as the matching. That proves that the matching is maximum, as from a previous theorem.

For a bipartite graph with a matching M, set x0 to be the set of M-unsaturated vertices in A and try to find an M-augmenting path incident with the vertices in x0

Set x = x0 (x ⊆ A) and y = ∅ (y ⊆ B) When you visit a vertex in your path, add the vertex to its appropriate set.

Eventually x = all the vertices in A reached (so far) by an M-alternating path originating in x0 and y = all the vertices in B reached (so far) by an M-alternating path originating in x0

Grow M-alternating paths as long as possible, that is until none of them can be grown or extended further.

Eventually, one of two things happens:

  1. There is a vertex in Y that is not M-saturated. Implies an augmenting path can be found.
  2. For every vertex a ∈ x, for every edge ab incident w/ a, b ∈ y. Implies that all vertices are M-saturated, which means a perfect matching between y and x \ x0. No edge connects x and B \ Y that is not in M. Such an edge would be a violation because it would extend an M-alternating path. Thus, the vertices in A \ X and Y form a cover of G. You should also recognize that the vertices in A \ X are not in x0 so they are all M-saturated. Si A \ X and B \ Y form their own matching, starting with y0 the M-unsaturated vertices in B. This would give you the same partitions.

Hall's Theorem

Let G be a bipartite graph with bipartition (A,B). There is a matching in G that saturates all the vertices in A iff for every subset S of A, |N(S)| ≥ |S|.
